关于度飞特跑路的维权记录

发表于 2018-06-16 | 分类于 杂事 |
首先我声明一点, 我不是度飞特的工作人员, 也不认识度飞特的老板和运营人员
  • 更新: 我觉得我们现在首先要做的事情是:
    1. 报案
    2. 选出一个代表来跟进

写这篇文章的目的, 是觉得微信群里面虽然人多, 但是说得比较乱, 没有人整理, 所以很多人都不想去看,而且群比较分散, 据我目前所知, 有两个微信群, 我只加了一个群–度飞特维权大润发分群, 另外一个群我不知道该如何加入. 所以我希望大家能到一个地方来讨论, 这篇文章会记录和群里面的一些重要信息, 并且会保持更新分享事件的进展, 另外文章最下方有留言板, 欢迎大家留言一起讨论.

我是大润发店的会员, 于2018年3月底办的两年会员卡, 度飞特大润发店于6月6日左右跑路的, 目前大润发店的大门上加上了三把锁, 好像是物业施工方和消费者加上的, 里面的健身器材都还在.

先说说我的维权经历:

  1. 6月8号打电话给了12315
  2. 6月14日南昌市工商局京东分局给我打电话来说这件案件涉及人数众多, 要我提交材料去分局.
  3. 我去了度飞特的总店, 在艾溪湖桥头的云中城A座负一层, 目前该店还在营业, 里面还是打着度飞特的招牌, 但是负责人员说他们和度飞特没有关系, 也不承认大润发店的会员, 不知道总店度飞特办的卡是不是可以继续使用, 欢迎你们在文章末尾留言告诉我.
  4. 然后我去了工商局京东分局, 据里面的警官说, 大润发店是无证经营, 所以现在合同的合法性比较难确认. 群里面说度飞特的老板叫李根, 我不知道他现在是被工商局控制着还是跑路了, 我所掌握的信息很少. 如果控制着相关人员还好说, 如果找不到人, 我们的钱应该很难被追回. 据我所知, 大润发店还欠着装修费十来万, 不知道是不是.

我个人说说我的想法:

  • 首先度飞特的事情肯定要继续追踪下去, 大家先去工商局报案, 报案人数越多, 涉事金额越大, 工商部门会越重视这个案件(文章下方有报案要准备的材料). 群里有人分享了一个12315的微信服务号, 我不确定这个上面报案是不是有一样的效果.
  • 我个人觉得度飞特跑路应该是没钱了, 资金链断裂了, 就算人找到了, 也很难拿回办卡的钱. 最多就是把运动器材找人处理换钱来赔偿, 但是这个钱应该远远不够赔偿吧.

作为消费者, 我只是想能够继续使用卡进行健身就行, 所以我想能不能把想继续健身的消费者统计一下数量(我希望大润发店这边有100人), 然后我们继续接下来进行运营, 前提是我们得了解度飞特现在的债务情况, 最好的情况是工商局出面来处理器材费用, 我们能优先接手运营.

我这里说的是众筹运营(最好有100人), 最好首先我们接下来别想着赚钱, 只要店里面接受的新会员能维持最低的人工费用, 房租水电, 器材折旧费用就行. 大家每个人都是股东, 希望大家能多去拉新会员, 我们没有盈利的目的, 只要能维持最低的运营费用就行, 这样会员费可以很低, 新会员也可以不用担心健身房跑路.

至于度飞特的债务问题, 我希望大家有律师朋友的, 可以去多问问要怎么处理. 我觉得最理想的状况是我们不用承担原先度飞特工资人员的拖欠工资以及装修施工方的欠款, 只是那器材处理获得的费用来偿还这些债务. 我觉得这些器材也不值什么钱, 今后运营的费用主要是房租那一块, 我不知道房租要多少钱, 我们来运营的话, 尽可能发动我们100个股东的力量来做日常维护, 比如开门, 打算卫生等等. 这些运营的细节是我们后面讨论的事情, 但我希望我们能开起一个我们自己的健身馆, 我就希望有一个早上7点钟就能运动的健身馆.

最后最重要的事情, 就是一定要去报案, 不能让坏人逍遥法外

分局地址在南京东路青春家园东门, 进了小区大门, 就是工商局. 进去之后大概走了半层, 左手边就是消费者12315接待部门, 联系电话 0791-88187235.
请先准备好相关材料:

  1. 办卡时的合同和收据,
  2. 自己手写或者打印好事情的经过, 包括办卡的经过, 办卡的费用, 损失的金额, 还有自己的联系方式
  3. 准备好度飞特的会员卡和你自己的身份证

请各位一定去报案, 这样我们消费者的损失金额越大, 人数越多, 赔偿起来, 我们分到的赔偿金越多, 你不去报案就一分钱没有

我希望, 就算我们不接手度飞特的店, 我们有相同爱好的也能开启一家众筹的全新的健身房.

欢迎大家在下面留言板讨论, 注意保留好个人的隐私信息.

fcp学习笔记

发表于 2018-05-20 |

** 快捷键

- `v`: 启用或者禁用片段
- `Fn + →`: 跳转到时间轴末端
- `Fn + ←`: 跳转到时间轴开头
- `j`, `k`, `l`: 反向播, 暂停, 正向播
- `k+l`, `j+l`: 慢速播放
- `/`: 播放截取的片段
- `i`, `o`: (in)截取clip的开始, (out)截取clip的结束
- `f`: 可以给clip或者clip的一部分加入favorite
- `del`: 可以给片段标记为隐藏
- `u`: 取消所有标记
- `w`: 将片段插入当前项目指针位置
- `e`: 将片段插入当前项目末端
- `x`: 在主线的某个片段上按`x`, 相当于在这个片段的开头和结尾按了i和o
- `shift + d`: 如果插入的片段比项目时间线长, 以片段的结尾对齐
- `shift + x`: 标记了时间标记后, 移动鼠标到想要删除的地方, 可直接删除时间标记到鼠标的部分
- `[`, `]`: 调整修正片断 
- `opt + [`, `opt + ]`: 裁剪片段上光标所在位置到片段前或者片段末尾
- `,`, `.`: 将项目的片段长度左右修正
- `,`, `.`: 如果选中的是整个片段操作, 将保持选中片段长度, 修改前后片段的长度
- `ctrl + d`: 显示片段的时长
- `shift + del`: 将项目的时间轴上选定部分删除并留空
- `shift + ?`: 播放附近片段
- `ctrl + +`, `ctrl + -`: 调整片段的音量
- `opt + 左键`: 在音轨上放上关键帧, 放上3个, 调节中间那个, 可以不影响外面的音轨
- `opt + g`: 将所选片段和音轨建立复合片段
- `y`: 试演
- `ctrl + ;`, `ctrl + '`: 在标记件前后切换
- `ctrl +,`, `ctrl + .`: 如果选中了标记, 需要先用上面的快捷键切换到标记上, 则是前后移动标记

** 如何让音轨不和第一个片段链接在一起
通常第一个片段会和音轨链接在一起, 可以选择除了音轨以外的所以片段, p后, 右移一些, 这样会在第一个片段前面创造出一个空白的片断, 再将音轨和这个空白片段链接在一起, 这样后面的片段就不会影响到音轨了.

** 人声一般在 -12 ~ -6 dB

** 多视角
在做多视角切割的时候, 如果已经做好的主线, 想要切换主线上的视角, 可以选中主线上的片段, 然后在想要切换的片段上面opt + 左键

** 批量修改静态图片时长
选中所有图片, ctrl + d后输入时长, 300代表3秒

** 转场特效

  • 选中主线上需要转场的片段, cmd + t
  • 光流转场特效可以缓解小部分剪切的卡顿

** 选中片段上的某一帧, 鼠标移动动片段上, 然后edit -> connect frezze frame, 可以把这一帧添加到主线上, 作为静态动画.

** 校色的时候, 一半让人色的明度在70,75左右

  20. 使用master代表分支master中最新的提交，也可以使用全称refs/heads/master或heads/master。

    使用HEAD代表版本库中最近的一次提交。

    符号^可以用于指代父提交。例如：

    • HEAD^代表版本库中的上一次提交，即最近一次提交的父提交。
    • HEAD^^则代表HEAD^的父提交。
      对于一个提交有多个父提交，可以在符号^后面用数字表示是第几个父提交。例如：
    • a573106^2的含义是提交a573106的多个父提交中的第二个父提交。
    • HEAD^1相当于HEAD^，含义是HEAD的多个父提交中的第一个父提交。
    • HEAD^^2的含义是HEAD^（HEAD父提交）的多个父提交中的第二个父提交。
      符号～＜n＞也可以用于指代祖先提交。例如：
      a573106～5即相当于a573106^^^^^。

  23. git reset --hard master@{2}, 找回reset前的状态

    git reset 详解

    两种格式
    用法一: git reset[-q][＜commit＞][--]＜paths＞...
    用法二: git reset[--soft|--mixed|--hard|--merge|--keep][-q][＜commit＞]
    第一种用法在命令中包含路径＜paths＞。为了避免路径和引用（或者提交ID）同名而发生冲突，可以在＜paths＞前用两个连续的短线（减号）作为分隔。
    第一种用法（包含了路径＜paths＞的用法）不会重置引用，更不会改变工作区，而是用commit下path的文件替换暂存区文件, 相当于取消之前的git add <path>命令对暂存区的修改

    • --hard: 执行1,2,3步, 替换后, 工作区,暂存区,和HEAD指向相同目录树
    • --soft: 执行1, 只改变引用指向, 不改变暂存区和工作区
    • --mixed: 这个是默认的, 执行1,2, 修改引用和暂存区, 不修改工作区
      示例:
      git reset, git reset --mixed HEAD, 修改引用和暂存区, 不修改工作区, 相当于git add的反向操作
      git reset -- filename, git reset HEAD filename: 将filename移除暂存区, 其他文件不动, 相当于git add filename的反向操作.
      git reset --soft HEAD^: 工作区和暂存区不变, 但是引用回退一次, 当对最新提交的提交说明或提交的更改不满意时，撤销最新的提交以便重新提交。git commit --amend里面用到了这个命令
      git reset (--mixed) HEAD^: 工作区不变, 引用和暂存区回退一次

  24. git show <commit>: 查看commit的详情

  25. git checkout: 如果不是checkout到分支, 将会导致detach状态(.git/HEAD里面是哈希值, 而不是分支名), 这个状态下提交的commit, 不能通过master或其他引用访问到(也就是切换回master时, 无法保持commit)

  26. 使用git merge修复detach.

    1. git branch -v: 确认现在在master分支.
    2. git merge <hashid>: hashid是detach状态下提交后的commit值, 这个值在切回master时会消失
    3. 合并后, 会出现detach模式下提交的文件

    深入git checkout

    这条命令会重写工作区
    用法一: git checkout[-q][＜commit＞][–]＜paths＞…
    用法二: git checkout[＜branch＞]
    用法三: git checkout[-m][[-b|–orphan]＜new_branch＞][＜start_point＞]

    与reset的不同

    第一种用法的＜commit＞是可选项，如果省略则相当于从暂存区（index）进行检出。这和上一章的重置命令大不相同：重置的默认值是HEAD，而检出的默认值是暂存区。因此重置一般用于重置暂存区（除非使用–hard参数，否则不重置工作区），而检出命令主要是覆盖工作区（如果＜commit＞不省略，也会替换暂存区中相应的文件）。

    第一种用法（包含了路径＜paths＞的用法）不会改变HEAD头指针，主要是用于指定版本的文件覆盖工作区中对应的文件。如果省略＜commit＞，则会用暂存区的文件覆盖工作区的文件，否则用指定提交中的文件覆盖暂存区和工作区中对应的文件。
    第二种用法（不使用路径＜paths＞的用法）则会改变HEAD头指针。之所以后面的参数写作＜branch＞，是因为只有HEAD切换到一个分支才可以对提交进行跟踪，否则仍然会进入“分离头指针”的状态。在“分离头指针”状态下的提交不能被引用关联到，从而可能丢失。所以用法二最主要的作用就是切换到分支。如果省略＜branch＞则相当于对工作区进行状态检查。

    例子:
    git checkout branch:
    checkout到branch分支, 完成图中1,2,3步骤, 更新HEAD指向branch, 用branch指向更新暂存区和工作区     git checkout, git checkout HEAD: 显示工作区, 暂存区与HEAD的差异
    git checkout -- filename: 用暂存区filename替换工作区对应文件, 相当于git add filename反向操作 git checkout branch -- filename: 保持HEAD不变, 用branch里面文件替换暂存区和工作区
    *git checkout ., git checkout -- .: 取消本地所有修改, 相当于用暂存区覆盖工作区
